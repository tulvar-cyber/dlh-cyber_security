## Permissions, SUID & SGID
Understanding Linux file permissions is fundamental for system security and access control. Permissions determine who can read, write, or execute files and directories, ensuring that users only access resources appropriate to their roles.

This module explores standard permission models in Unix/Linux systems, including ownership (user, group, others) and permission types (read, write, execute). It also introduces special permission bits such as SUID (Set User ID) and SGID (Set Group ID), which allow executable files to run with elevated privileges under specific conditions.

While these mechanisms are essential for legitimate system operations, improper configuration can create serious security vulnerabilities. Therefore, understanding how permissions, SUID, and SGID work is critical for both system administration and cybersecurity professionals, especially in privilege escalation analysis and forensic investigations.
